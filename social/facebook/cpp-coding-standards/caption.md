# C++ coding standards in 2026

## Body
Three coding standards every regulated C++ project will encounter: MISRA C++:2023 (the AUTOSAR C++14 successor, automotive/embedded), SEI CERT C++ (security-focused, government and finance), and JSF AV C++ (older, but still cited in defence avionics work).

The wro.cpp page maps them side by side: what each rule actually checks, which overlap, which are unique to one standard, and how to compose them in a single clang-tidy configuration.

Bonus: the reflection-driven composable rule bundle. Annotate types with rule sets they must comply with; the lint runs at compile time, surfaces violations with field names, and shrinks the surface area your auditor has to review.
