# The July 2026 C++ mailing, in five papers

## Body
The first WG21 mailing after the June Brno meeting is out, and it is mostly about memory safety. Four papers argue the same question: what is a safety profile.

Peter Bindels writes three papers (P4314 to P4316) pinning down how you activate a profile and what its spec must contain. Vinnie Falco proposes std::core_ub (P4317), a profile that turns catchable core-language UB into a defined trap. Bjarne Stroustrup's initialization profile reaches R2 (P4222). Then Falco again, in P4318, argues that implicit runtime checks in the core language cost more than they save. The same author proposing a checking profile and arguing against implicit checks is the interesting part.

The wildcard: P4312 "Effect Sets" generalizes noexcept into composable effect annotations.
